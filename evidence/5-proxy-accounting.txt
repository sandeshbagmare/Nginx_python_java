# Evidence 5 — the proxy's own header accounting (one line = the whole bug)

NAIVE proxy (java-proxy/NaiveProxy.java) stdout:
  [NAIVE] copied-TE-from-upstream=1 + framing-TE=1  => Transfer-Encoding lines on wire=2

FIXED proxy (java-proxy/FixedProxy.java) stdout:
  [FIXED] copied-TE-from-upstream=0 + framing-TE=1  => Transfer-Encoding lines on wire=1

Two INDEPENDENT code paths each contribute a Transfer-Encoding header:
  (1) copying the hop-by-hop header from the upstream response  <- THE BUG  (NAIVE=1, FIXED=0)
  (2) the proxy server's own chunked framing of the body         <- always 1 (correct)
NAIVE = (1)+(2) = 2 headers -> NGINX 502 ;  FIXED = (0)+(2) = 1 header -> NGINX 200
